# Свадебный сайт-приглашение 💍

Одностраничный статический сайт: у каждой группы гостей — своя персональная ссылка
с личным приветствием, сообщением, фотографией и собственным оформлением (цвета, шрифт, фон).

## Файлы проекта

| Файл | Назначение |
|---|---|
| `index.html` | Страница-каркас, подключает стили и скрипты |
| `style.css` | Всё оформление (адаптивное, мобильная версия) |
| `script.js` | Логика: разбор `?guest=`, загрузка данных, отрисовка страниц |
| `guests.json` | **Основные данные**: блок `wedding` (общая информация) + список гостей |
| `data.js` | Автокопия `guests.json` — чтобы сайт работал при открытии файла без сервера |

## Как запустить локально

**Самый простой способ** — двойной клик по `index.html` (сработает благодаря `data.js`).

**Правильный способ** (как на хостинге, через `guests.json`):
- VS Code → расширение **Live Server** → правый клик по `index.html` → *Open with Live Server*, или
- в PowerShell из папки проекта: `python -m http.server 8000` и открыть `http://localhost:8000`.

## Что обязательно заменить перед отправкой гостям

1. **Блок `wedding` в начале `guests.json`** — сейчас там заглушки:
   имена жениха и невесты, дата (`dateText`), время, место (`venue`, `address`), хештег.
2. **`rsvpLink`** у каждого гостя — поставьте ссылку на вашу Google-форму или Telegram.
   Можно одну и ту же для всех (поиск-замена `https://forms.gle/ВАША_ФОРМА`).
3. **`photo`** — у каждой группы стоит своя фотография цветов (с Викисклада, вид цветка подобран
   под цветовую тему). Если хотите реальные фото гостей — замените ссылку
   (или положите файлы в папку `photos/` и укажите путь вида `photos/roman-tatyana.jpg`).
4. **`backgroundImage`** — у каждого свой нежно-свадебный фон (букеты, кольца, цветение, ленты).
   Тоже можно заменить на свои. По желанию поменяйте и тексты `message`.

> ⚠️ **После любого изменения `guests.json` пересоздайте `data.js`** (он нужен для работы без сервера).
> В PowerShell из папки проекта:
> ```powershell
> $json = [IO.File]::ReadAllText("guests.json", [Text.Encoding]::UTF8); [IO.File]::WriteAllText("data.js", "window.GUESTS_DATA = $json;", (New-Object Text.UTF8Encoding $false))
> ```

## Размещение на GitHub Pages (бесплатный хостинг)

1. Создайте репозиторий на GitHub (например, `wedding`).
2. Загрузите туда все файлы проекта (через сайт GitHub: *Add file → Upload files*).
3. В репозитории: **Settings → Pages → Source: Deploy from a branch**, ветка `main`, папка `/ (root)` → Save.
4. Через пару минут сайт будет доступен по адресу
   `https://ВАШ_ЛОГИН.github.io/wedding/`,
   а ссылка гостя — например `https://ВАШ_ЛОГИН.github.io/wedding/?guest=roman-tatyana`.

Подойдёт и любой другой статический хостинг (Netlify, Vercel, Cloudflare Pages) — просто
загрузите папку целиком, ничего собирать не нужно.

## Как добавить или изменить гостя

Скопируйте любой блок в массиве `guests` в `guests.json` и поменяйте:
`id` (латиницей, уникальный), `name`, `salutation` (Дорогой/Дорогая/Дорогие), `message`,
`photo`, `rsvpLink` и тему (`primaryColor`, `backgroundColor`, `fontFamily`, `backgroundImage`,
`accentImage` — ссылка на картинку-акцент, или `accentEmoji` — просто эмодзи).
Не забудьте пересоздать `data.js` (команда выше).
